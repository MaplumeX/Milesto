# 任务编辑纸面亮度微调设计

日期：2026-03-16
状态：已确认，待进入实现计划

## 1. 背景

上一轮夜间模式调整后，内容区整体方向已接近目标，但任务编辑组件的纸面层仍然显得比内容区更深，导致编辑态没有被清楚地读成“浮在内容区上方的纸面”。

这次调整只解决一个问题：让任务编辑纸面比内容区更浅，但不把整个面板体系一起抬亮。

## 2. 已确认结论

用户已确认以下范围与方向：

- 目标：任务编辑纸面应当比内容区更浅
- 适用范围：同时覆盖内联编辑纸面和覆盖式编辑纸面
- 不调整范围：编辑相关 `popover` 不跟随变浅
- 亮度档位：采用平衡抬亮方案（B 档）

目标编辑纸面色：

- `editor paper`：`#31353A`

现有层级关系保持为：

- 内容区：`#222528`
- 编辑纸面：`#31353A`
- 普通 `popover`：保持当前表面层颜色，不提升到编辑纸面层

## 3. 非目标

本次不做以下变更：

- 不调整侧边栏、内容区、底部栏颜色
- 不调整普通卡片、输入框、详情面板颜色
- 不调整 `task-inline-popover`
- 不修改任务编辑组件结构或 JSX

## 4. 设计原则

### 4.1 KISS

只新增一个专用 token，并让两个编辑纸面选择器消费它，不重做现有表面层体系。

### 4.2 DRY

不分别给 `task-inline-paper` 和 `overlay-paper` 写两套色值，统一走共享的 `--editor-paper`。

### 4.3 YAGNI

不引入新的主题对象、不新增多档编辑纸面变体，也不顺手调整 `popover`。

## 5. 方案对比

本次评估了三档编辑纸面亮度：

### 5.1 方案 A：轻抬一层

优点：

- 最保守
- 对现有观感冲击最小

缺点：

- 纸面感仍然偏弱

### 5.2 方案 B：平衡抬亮（采用）

优点：

- 能明确读出编辑态是浮在内容区上的纸面
- 不会亮到像独立弹层
- 与当前暖石墨主题兼容性最好

缺点：

- 需要新增专用 token，避免影响普通表面层

### 5.3 方案 C：更强纸面感

优点：

- 编辑态辨识度最高

缺点：

- 容易显厚
- 会开始接近弹层，而不是编辑纸面

## 6. 实现设计

### 6.1 文件范围

本次实现只应修改：

- `src/index.css`

原则上不改 `TaskEditorPaper.tsx` 结构，也不改 Electron/Main 逻辑。

### 6.2 Token 设计

在主题 token 中新增编辑纸面专用变量：

- `--editor-paper`

暗色主题下赋值为：

- `--editor-paper: #31353A`

浅色主题可先回退到现有 `--paper`，避免引入无关浅色改动。

### 6.3 选择器落点

以下选择器应改为使用 `--editor-paper`：

- `.task-inline-paper`
- `.overlay-paper`

以下选择器保持不变：

- `.task-inline-popover`
- `.card`
- `.input`
- `.detail`

### 6.4 层级语义

调整后的语义应该是：

- 内容区负责页面基底
- 普通表面层负责常规表单/卡片/popover
- 编辑纸面单独抬高半层，只服务于任务编辑态

这样可以保证“编辑态更突出”，但不会把所有次级表面层一起拉亮。

## 7. 验证方案

### 7.1 必做验证

- `npx tsc -p tsconfig.json`
- `npm test`

### 7.2 最小主题护栏

应补充或调整断言，确认深色主题下：

- `task-inline-paper` 的背景色为编辑纸面色
- `overlay-paper` 的背景色为同一编辑纸面色
- `task-inline-popover` 不等于编辑纸面色

### 7.3 人工观感检查

重点检查：

- 内联编辑卡片是否明显比内容区更浅
- 覆盖式编辑纸面是否与内联编辑属于同一层级
- `popover` 是否仍然比编辑纸面更“普通”，没有一起跳亮

## 8. 风险与回退

主要风险：

- 编辑纸面仍然不够浅，层级感不够
- 编辑纸面过亮，开始像弹层

回退方式很简单：

- 只回调 `--editor-paper`
- 不需要回退组件结构或其他 token
