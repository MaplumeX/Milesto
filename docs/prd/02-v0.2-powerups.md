# Milesto PRD - v0.2（Power-ups: 提醒/重复/组织增强）

版本: v0.2
状态: Done
前置: v0.1（单机闭环）
目标周期: 4-6 周（建议）

## 1. 迭代目标

- 把 Milesto 从“能用”推进到“好用”: 用提醒、重复与更强的组织能力减少遗忘与维护成本。
- 保持低摩擦: 强能力不牺牲输入与日常执行速度。

## 2. 范围定义

### 2.1 In Scope

- 提醒（单次提醒）
- 重复任务（按日期/按完成）
- 项目分组（Sections/Headings）
- 过滤能力增强（基于 due/scheduled/归属）
- Checklist 体验增强

### 2.2 Out of Scope

- 多端同步
- 端到端加密
- 高级自动化/插件系统

## 3. 用户故事

1. 作为用户，我能给任务设置提醒，不再错过关键事项。
2. 作为用户，我能创建重复任务（例如每周回顾/每月缴费），并信任它可靠生成。
3. 作为用户，我能在项目里用分组组织任务，让长项目更可读。
4. 作为用户，我能通过过滤快速找到“即将到期/没有归属/没有计划”的任务进行清理。

## 4. 功能需求与验收标准

### 4.1 提醒（Reminders）（P0）

- 任务级提醒
  - 模型: reminder_at（单一时间点）或 reminders[]（可扩展）
  - 验收: 到达 reminder_at 时触发提醒（触发形态取决于平台; 至少在应用内可见）。
- 提醒管理
  - 验收: 用户可新增/修改/删除提醒; 删除后不再触发。

### 4.2 重复任务（Repeat Rules）（P0）

- 重复规则类型（最小集合）
  - 每日/每周/每月
  - 按日期触发（例如每周一生成）
  - 按完成触发（例如完成后 7 天再生成）
  - 验收: 规则可创建、可编辑、可停用。
- 生成策略（重要）
  - 验收: 不重复生成（同一规则在同一周期只生成一次）；生成结果可追溯到规则来源。

### 4.3 Projects 分组（Sections/Headings）（P0）

- 项目内分组
  - 验收: 项目可创建多个 section; 任务可移动到 section; section 内可排序。

### 4.4 过滤与清理（P0）

- 过滤器（建议最小集合）
  - due 在 X 天内
  - 无 project 的任务
  - 无 scheduled 的任务
  - 验收: 过滤器可组合（至少与标签可组合）。

### 4.5 Checklist 增强（P1）

- checklist item 的快捷创建/完成
  - 验收: 在任务详情中可快速添加多条 checklist; 逐条勾选不影响主任务完成状态（或由规则决定，但必须一致）。

## 5. 体验与非功能（本迭代重点）

- 不打扰: 提醒必须可静默/可关闭。
- 可解释: 重复任务的来源必须可见（例如显示由某规则生成）。

## 6. 风险与对策

- 重复任务 bug 容易造成任务爆炸: 需要幂等生成与可追溯机制。
- 提醒跨平台实现差异: 先保证应用内提醒可用，再逐步接系统级通知。

## 7. 交付清单

- 单次提醒可用
- 重复任务可用且稳定
- 项目分组可用
- 过滤能力增强
