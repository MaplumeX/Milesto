# Milesto PRD - 产品概览

版本: v0.1
状态: Done
日期: 2026-01-25

## 1. 背景与愿景

Things 3 代表了个人 GTD/待办应用在“低摩擦输入 + 清晰信息架构 + 流畅交互”上的高水准，但它是闭源且生态受限（平台、数据/同步可控性、可扩展性）。

Milesto 的愿景是成为一个面向个人用户的开源替代品：覆盖“收集-整理-执行-回顾”的核心闭环，同时提供更强的数据主权、可扩展性与跨平台能力。

## 2. 产品目标与成功指标

### 2.1 产品目标

- 提供个人任务管理系统，覆盖从快速收集到日常执行的完整闭环。
- 默认本地优先（local-first），离线可用，数据可导出，长期可迁移。
- 键盘优先与高性能：常用操作在 1-2 次键盘交互内完成。
- 可持续开源协作：模块边界明确，易于贡献。

### 2.2 成功指标（可量化）

- 新用户在 10 分钟内完成: 创建 1 个项目、录入 10 个任务、完成 1 次 Today 规划。
- 10k 任务规模下: 搜索首屏结果 < 200ms；列表滚动无明显卡顿。
- 数据可靠性: 无已知导致数据丢失的缺陷；冲突合并可解释、可回滚。

## 3. 非目标（首阶段不做）

- 多人协作/团队项目管理。
- 复杂工时、成本、甘特图等 PM 套件能力。

## 4. 用户与场景

### 4.1 目标用户

- GTD 重度用户: Inbox 清空习惯，依赖 Today/Upcoming 做编排。
- 轻量待办用户: 希望比便签更系统，但不想承担高学习成本。
- 隐私/数据主权用户: 需要本地存储、可导出、可自建同步。

### 4.2 关键场景

1. 快速收集: 2 秒内添加任务（支持快捷键/命令面板）。
2. 日计划: 在 Today 快速挑选并排序今天要做的任务。
3. 项目推进: 项目内存在明确下一步（Next Action），支持分组/清单。
4. 未来安排: Upcoming 浏览未来计划任务与截止日期。
5. 回顾清理: 定期查看 Someday/Anytime，避免遗忘与堆积。

## 5. 产品原则

- Local-first: 默认无需账号即可使用；同步是可选能力。
- 低摩擦: 输入、整理、完成路径尽可能短。
- 可解释: 自动化结果可追溯（例如重复规则、冲突合并）。
- 不打扰: 通知与提醒可控、可静默。
- 开源友好: 模块化、可测试、可替换（存储/同步/通知等）。

## 6. 信息架构

- Inbox
- Today
- Upcoming
- Anytime
- Someday
- Logbook（已完成/归档）
- Projects
- Areas（领域/责任区，可选）
- Tags
- Search
- Settings

## 7. 核心对象（概念模型）

### 7.1 Task（任务）

- id
- title
- notes（支持 Markdown）
- status: open | done | cancelled
- project_id（可空）
- area_id（可空）
- tag_ids（可空）
- scheduled_at（计划出现/开始日期; 决定它何时出现在 Today/Upcoming）
- due_at（截止日期; 用于提醒与风险标识）
- checklist_items: [{ id, title, done }]
- created_at / updated_at / completed_at
- order（列表内排序）

### 7.2 Project（项目）

- id
- title
- notes
- area_id（可空）
- status: active | on_hold | done
- sections（分组/标题）

### 7.3 Area（领域）

- id
- title
- notes

### 7.4 Tag（标签）

- id
- title
- color（可选）

### 7.5 Reminder / RepeatRule（后续迭代）

- reminder: 任务级提醒（时间点）
- repeat_rule: 每日/每周/每月/自定义（按完成或按日期触发）

## 8. 非功能需求（全局约束）

- 性能:
  - 冷启动至可交互 < 1s（桌面/本地环境目标值）
  - 搜索 < 200ms（10k 任务规模）
- 可靠性:
  - 写操作具备事务性; 至少保证不会悄悄丢数据
  - 同步冲突必须可见、可手动处理（v0.3）
- 可维护性:
  - 核心逻辑与 UI 解耦
  - 关键路径有自动化测试覆盖
